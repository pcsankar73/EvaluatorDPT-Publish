# Security Policy

## Reporting a Vulnerability

If you believe you have found a security or privacy issue in this dataset repository (for example: leaked secrets, PII exposure, malicious files, or supply-chain risk in scripts):

1. Do **not** open a public issue with sensitive details.
2. Contact the maintainers privately and include:
   - a clear description of the issue,
   - steps to reproduce or locate the problem,
   - the affected file paths and commit hash (if known).

## Scope

This policy covers:
- Repository contents (documentation, scripts, configuration)
- Packaged artifacts committed to the repository (including data files)

This policy does not cover:
- Third-party infrastructure or services not controlled by the maintainers
