════════════════════════════════════════════════════════════════════════════════════════════════════
  FULL DATASET LINEAGE DETAIL
════════════════════════════════════════════════════════════════════════════════════════════════════

── DS-B (clean) ─────────────────────
  Description: Full collected corpus — all available labelled data after noise removal
  Role: Starting point for all modelling. Establishes the full task distribution.
  Splits: train=651,240  val=81,372  test=81,384  total=813,996
  Quality actions:
    - Removed sentiment140, imdb, empathetic_dialogues (sentiment→label corruption)
    - Removed short texts <5 tokens (~43K rows)
    - Fixed HTML/encoding artifacts (~9K rows)
    - Upgraded emotion/value vectors from class-level to per-sample (100% coverage)
  Notes: DS-B noisy (820K) was the intermediate; DS-B clean (814K) is the stable corpus used going forward.

── DS-E (NLI-only) ─────────────────────
  Description: NLI-filtered subset: only entailment/contradiction pairs retained
  Role: Key refinement: removed non-NLI rows that mapped polarity sentiment to YES/NO, corrupting the decision boundary. This was the biggest single F1 jump (+0.033).
  Splits: train=355,408  val=44,404  test=44,597  total=444,409
  F1 impact: +0.033 vs DS-B (largest single-step improvement in programme)
  Quality actions:
    - Retained only textual entailment/contradiction pairs
    - Excluded sentiment/opinion-only sources
  Notes: Volume reduction 814K→444K was intentional: better task fit, not data loss.

── DS-F ─────────────────────
  Description: DS-E + 3,462 TBD→NO label corrections
  Role: Label quality pass. These were NLI pairs where hypothesis was clearly false (contradicted by premise) but had been labelled TBD. Correcting them sharpened the NO boundary.
  Splits: train=355,408  val=44,404  test=44,597  total=444,409
  F1 impact: +0.013 vs DS-E
  Quality actions:
    - 3,462 TBD→NO label corrections
  Notes: Stable text+label baseline. All subsequent datasets share this text.

── DS-G ─────────────────────
  Description: DS-F text + diversity-corrected aux vectors (value dimension bug fix)
  Role: Corrected a value-vector dimension bug that caused collapsed diversity in the V10 head. Same texts and decision labels as DS-F.
  Splits: train=355,408  val=44,404  test=44,597  total=444,409
  Quality actions:
    - 100% emotion binary diversity (target >90%)
    - 100% value binary diversity
    - Corrected value dimension encoding bug
  Notes: Text and decision labels unchanged from DS-F.

── DS-J (canonical, read-only) ─────────────────────
  Description: DS-G + hypothesis filter + correct value dims + emotion alignment fix. Final consolidated dataset for this lineage.
  Role: Canonical dataset — all subsequent datasets build from DS-J CSVs. Val and test splits are permanently fixed here.
  Splits: train=355,408  val=44,404  test=44,597  total=444,409
  Quality actions:
    - Hypothesis-only filter applied (removes trivially predictable pairs)
    - Emotion label alignment fix
    - Value dimension correction confirmed
    - Cross-split leakage: 0 rows
  Notes: Val=44,404 and Test=44,597 rows are the fixed evaluation splits for all subsequent stages including DS-L. DS-J train is the text/label base for DS-L train.

── DS-L (current training set) ─────────────────────
  Description: DS-J base + DS-K unique hard cases + capped reasoning/negation pattern boost. Masks: emotion_mask=0.0 (skip emotion supervision), value_mask=0.3 (weak).
  Role: Primary training dataset for S12 and S12B. Adds targeted hard cases (negation, numbers, role/value swaps) using capped targeted hard-case additions.
  Splits: train=418,725  val=44,404  test=44,597  total=507,726
  Notes: Val and test splits are identical to DS-J (build script writes dsj_val/dsj_test unchanged). Train is DS-J train + DS-K unique rows + capped boost (max 2 duplicates per text). Emotion head intentionally masked (emotion_mask=0) — E28 metrics skipped.

── BP-S12 (boundary pack — correction only) ─────────────────────
  Description: 323 training examples mined from S12 ep3 diagnostics. Used for 1 targeted sharpening epoch after S12 ep3 checkpoint.
  Role: Targeted correction of high-confidence boundary errors. NOT the primary training dataset — represents <0.1% of total training volume. Sourced from: high_conf_errors, hardest_examples, disagreement_audit from S12 ep3 5k-slice diagnostic run.
  Splits: train=323  val=500  test=500
  Notes: S12B FULLVAL/FULLTEST evaluation was run against DS-L val/test (44,404/44,597 rows), NOT against the boundary pack val/test. The boundary pack val/test (500 rows each) were used only during the 1-epoch sharpening training run for checkpoint selection.
