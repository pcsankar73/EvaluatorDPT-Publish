  SECTION 1: DECISION HEAD — LABEL DEFINITIONS (YES / NO / TBD)
════════════════════════════════════════════════════════════════════════════════

  The decision head classifies whether a hypothesis is supported by a premise.
  This is a Natural Language Inference (NLI)-derived three-way classification:

  Class 0 — YES (Entailment):
    The hypothesis is clearly and directly supported by the premise.
    The premise provides sufficient evidence to conclude the hypothesis is true.
    Example:
      Premise:   "All employees received a 10% pay raise this year."
      Hypothesis:"Employee salaries increased this year."
      Label:     YES — the premise entails the hypothesis directly.

  Class 1 — NO (Contradiction):
    The hypothesis is clearly contradicted by the premise.
    The premise provides direct evidence the hypothesis is false.
    Example:
      Premise:   "The company reported zero profit for the quarter."
      Hypothesis:"The company was profitable this quarter."
      Label:     NO — direct contradiction.

  Class 2 — TBD (Uncertain / Insufficient Evidence):
    The hypothesis is neither clearly supported nor clearly contradicted.
    This includes cases where:
      (a) The premise is silent on the hypothesis (neutral / irrelevant)
      (b) The evidence is partial — supports or contradicts somewhat but not conclusively
      (c) The relationship requires world knowledge or inference beyond the premise
      (d) The hypothesis is ambiguous relative to the premise
    Example:
      Premise:   "The manager approved one new project this quarter."
      Hypothesis:"The manager approved all new projects this quarter."
      Label:     TBD — premise mentions one project, does not confirm or deny 'all'.

  SECTION 2: TBD LABEL POLICY — HOW TBD LABELS WERE ASSIGNED
════════════════════════════════════════════════════════════════════════════════

  Source of labels:
    Labels are derived from the NLI task formulation, not from a custom human
    annotation campaign. The three source datasets (SNLI, MultiNLI, SemEval NLI)
    each provide three-way labels: entailment / contradiction / neutral.

    Mapping applied:
      entailment   → YES
      contradiction → NO
      neutral       → TBD

  This means TBD is a policy-derived label: it reflects the NLI dataset's
  "neutral" category, which was defined by the original dataset authors via
  crowdsourced annotation with majority-vote adjudication.

  IMPORTANT — Human IAA (Inter-Annotator Agreement):
    No independent human IAA study was conducted by this project.
    The TBD labels inherit the IAA of the source NLI datasets:
      - SNLI: reported κ ≈ 0.70 (Bowman et al., 2015)
      - MultiNLI: reported κ ≈ 0.71 (Williams et al., 2018)
      - SemEval NLI: task-specific annotation guidelines
    These are cited as the ground-truth annotation quality for the decision labels.
    Any future human re-annotation or IAA study would be additional validation.

  Label corrections applied during dataset refinement:
    DS-F: 3,462 TBD→NO corrections
      These were cases where the hypothesis was clearly contradicted by the
      premise (direct factual negation) but had been labelled TBD by the
      original annotators. These corrections were applied via rule-based
      heuristics (hypothesis-premise contradiction patterns) and manually
      reviewed. No new human annotation was commissioned.
    Source: docs/programme/01-dataset-registry.md, DS-F section.

  Hypothesis filter (DS-J):
    A hypothesis-only filter was applied to remove trivially predictable pairs
    (where the hypothesis alone, without the premise, would allow a classifier
    to predict the label with high confidence). This is a data quality technique
    documented in docs/programme/07-data-quality-techniques.md.

  SECTION 3: AUX HEAD LABELS — EMOTION (E28) AND VALUE (V10)
════════════════════════════════════════════════════════════════════════════════

  Emotion labels (E28 — 28-dimensional soft vector):
    Sourced from GoEmotions-aligned annotation (Demszky et al., 2020).
    Labels are soft probabilities (not hard 0/1), derived from multi-label
    crowdsourced ratings. Per-sample vectors, not class-level averages.
    Format: float in [0, 1] per emotion dimension.
    Mask policy in DS-L: emotion_mask = 0.0 for ALL rows.
      → Emotion supervision is entirely disabled in DS-L and S12/S12B training.
      → Emotion metrics are intentionally skipped in all DS-L stage reports.
      Reason: Focus on improving decision head F1. Emotion head will be
      re-enabled in a future stage once decision target is met.

  Value labels (V10 — 10-dimensional soft vector):
    Sourced from ethics/values dataset annotation (commonsense ethics,
    virtue ethics). Soft probabilities per value dimension.
    Mask policy in DS-L: value_mask = 0.3 for ALL rows.
      → Value supervision applies with 30% weight (weak supervision).
      → Value head metrics are reported but treated as secondary.
    Confirmed Value F1 (DS-L test): ~0.444 (macro F1, masked evaluation).

════════════════════════════════════════════════════════════════════════════════
  SECTION 4: MASK POLICY SUMMARY
════════════════════════════════════════════════════════════════════════════════

  Masks control per-sample supervision weight for aux heads:

  emotion_mask:
    0.0 in DS-L → emotion loss term = 0 for all samples
    Rationale: emotion head was not the focus of S12/S12B. Disabling it
    prevents noisy emotion gradients from interfering with decision head.

  value_mask:
    0.3 in DS-L → value loss weight = 0.3 for all samples
    Rationale: weak value supervision maintained to prevent value head collapse,
    while keeping decision head as primary training signal.

  decision_label: always fully supervised (no mask).
